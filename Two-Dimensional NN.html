findinterceptslope<-function(w)
{
  slope<-w[2]/w[3]*(-1)
  intercept<-w[1]/w[3]*(-1)
  rvector<-c(intercept,slope)
  return(rvector)
}
for(jj in 1:nrepeats)
{
  plot(train$X1,train$X2,col=c("black","red")[train$Label+1],xlim=c(-2,2),ylim=c(-2,2))
  weights<-net$weights[[jj]][[1]]
  for(ii in 1:nhiddens)
  {
    node.weights<-weights[,ii]
    line<-findinterceptslope(node.weights)
    abline(line[1],line[2],col="green",lty=2)
  }
}

findinterceptslope<-function(w)
{
  slope<-w[2]/w[3]*(-1)
  intercept<-w[1]/w[3]*(-1)
  rvector<-c(intercept,slope)
  return(rvector)
}

Each hidden node in a multi-layer neural network is used to capture the complexity of the input data. An increase in hidden nodes will result in a more complex decision region. It will also increase the computational complexity of training the network, as well as the amount of training data necessary to train it. In our case, since there are only 2 inputs, a hidden layer with two or fewer hidden nodes is ideal. The output with three hidden nodes has led to an overfitted model, and is too complex. Also, since we only have 9 training instances, there is likely not enough data with which to train a more complex network.
